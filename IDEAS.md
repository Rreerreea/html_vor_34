---

## TL;DR — на чём играем

Конверсия «URL → Figma» **коммодитизируется**: у Figma теперь свой Dev Mode MCP в режиме записи, у html.to.design — свой MCP-сервер. «Вставил ссылку — получил макет» больше не моат. Моат сместился на **чистоту слоёв + токены + приватность + цену**.

Мы не догоняем лидера по точности (у них 1.2M юзеров и 4.7/5). Мы занимаем три ниши, которые **облачная модель закрыть структурно не может**:

1. **РФ / Индия** — html.to.design им недоступен (индивидуальный биллинг не работает, футболят в «Teams / contact sales»).
2. **Приватность / NDA / localhost / staging** — облако рендерит на своём сервере; мы — на машине пользователя.
3. **«Восстановить дизайн с живого сайта, когда исходников нет»** — редизайны, аудиты, разбор конкурентов.

> Позиционирование одной фразой: **бесплатный, локальный, открытый html.to.design, который выдаёт чистые редактируемые слои и Figma Variables.**

## Идеи по приоритету

### Tier 1 — киллер-дифференциаторы (дёшево, бьют в их дыры)

**1.1 Чистые слои by construction** 🎯 _боль №1_
- **Зачем:** мусорные слои — самая подтверждённая жалоба по всем инструментам; существует целая индустрия cleanup-плагинов, которые чистят ПОСЛЕ. Мы делаем чисто СРАЗУ.
- **Как:** в сериализаторе помечать «несёт ли узел визуал» (фон/бордер/тень/радиус/клип/картинка). При построении схлопывать контейнеры с одним ребёнком и без визуала (поднять ребёнка, сложив координаты), выкидывать пустые фреймы. Осмысленные имена слоёв: по тегу (`header`, `nav`, `button`), по тексту («Купить»), по `aria-label`/`alt`/`role` — вместо «div».
- **Усилие:** средне. Самый заметный «вау» на демо.

**1.2 Figma Variables + экспорт W3C-токенов** 🎯 _никто не делает_
- **Зачем:** токены — стандарт (W3C DTCG стабилен с окт-2025), а почти все экстракторы выдают хардкод-значения. «Достать токены с сайта» востребовано и не занято.
- **Как:** на сервере собрать частотные значения (цвета фон/текст/бордер, размеры/начертания шрифта, отступы, радиусы, тени) → дедуп в словарь. В плагине создать Variable Collection «Imported tokens» и привязать заливки/текст к переменным. Кнопка «Экспорт tokens.json» (формат DTCG). Если снимаем светлую+тёмную тему → Variable Modes.
- **Усилие:** средне.

**1.3 Приватность как фича** 🎯 _их структурная дыра_
- **Зачем:** аудируемая гарантия, что дизайн-данные не покидают машину — то, чего cloud дать не может.
- **Как:** залочить в манифесте `networkAccess.allowedDomains` на `["http://localhost:3789"]` вместо `["*"]` (картинки и так приходят base64 через локальный сервер — внешние домены не нужны). Для auth/staging — приём `storageState` (cookies/сессия) или подключение к запущенному Chrome через CDP. В README — явно про текст-как-текст и локальность.
- **Усилие:** низко (lock) + средне (auth-режим).

### Tier 2 — закрыть критичные гэпы (догон там, где разрыв убивает ценность)

**2.1 CSS flex/grid → нативный Auto Layout** _боль №2, наш слабейший пункт_
- **Зачем:** «выглядит правильно, но разваливается при правке» — главная претензия к импортёрам. Без Auto Layout макет не редактируется по-человечески.
- **Как:** прокинуть с сервера `display`, `flex-direction`, `gap`, `justify/align`, `padding`. В плагине для flex-контейнера включать `layoutMode` H/V, `itemSpacing`, паддинги, выравнивания — вместо абсолютных координат. Только когда дети реально в потоке. Поэтапно: сначала простые ряды/колонки, grid позже. Безопасный фоллбэк на абсолют, если уверенности нет.
- **Усилие:** высоко, поэтапно. Самая большая инженерная инвестиция.

**2.2 Детект подменённых шрифтов**
- **Как:** собирать карту «запрошенный шрифт → применённый Inter», в конце показывать «Заменены шрифты: X → Inter (N слоёв)» + подсказку установить. У них one-click download — у нас хотя бы прозрачность вместо тихой замены.
- **Усилие:** низко-средне.

**2.3 Мульти-вьюпорт за один проход**
- **Как:** в UI чекбоксы пресетов (1920 / 1440 / 768 / 390), сервер снимает каждый, плагин строит фреймы рядом. Один клик — desktop + mobile. Параметр ширины уже есть.
- **Усилие:** низко.

### Tier 3 — стратегические ставки (тренды, будущее)

**3.1 Свой локальный MCP-сервер**
- **Как:** обернуть `/capture` в MCP-инструмент `import-url` → Claude/Cursor смогут «положи сайт X в Figma локально». Future-proof против коммодитизации + история приватности (это твой локальный MCP, не облачный). Сервер уже готов.
- **Усилие:** низко-средне.

**3.2 OSS-релиз**
- **Как:** выложить на GitHub (MIT/AGPL), оформить README, позиционировать как **поддерживаемый преемник** заброшенного BuilderIO/figma-html. Захват сегмента «без закрытого облака, хочу форкнуть».
- **Усилие:** низко (упаковка).

**3.3 (опц.) Design-vs-shipped QA / overlay-diff**
- Наш рендер-движок может питать сравнение макета с живым/localhost-билдом (наложение, пиксельный diff). Растущая ниша в эпоху Dev Mode. Потенциально отдельный продукт.

## Чего НЕ делать

- ❌ Гнаться за пиксель-перфект fidelity в лоб — у них 4.7/5 и реальные фанаты.
- ❌ Маркетить «они плохие» — «быстрее с нуля» это спорный тезис, не консенсус.
- ❌ Radial/conic градиенты, полный round-trip (Figma→код) — отложить.

**Топ-3 дешёвых и точных:** `1.1 Чистые слои` → `1.2 Variables/токены` → `1.3 заявить приватность + текст-как-текст`.

Моя ставка на первый заход — **1.1 Чистые слои**: бьёт в жалобу №1, виден результат на любом импорте, не требует новой инфраструктуры.
