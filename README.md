# HTML to Figma (Free)

Бесплатный self-hosted аналог html.to.design. Импортирует любой сайт в Figma
как редактируемый дизайн — без подписки, всё крутится локально на твоей машине.

## Как это работает

```
URL ──▶ [локальный сервер: Chromium рендерит страницу] ──▶ JSON (дерево + стили + картинки)
                                                              │
                              [Figma-плагин строит ноды] ◀────┘
```

Сервер делает то, за что html.to.design берёт деньги: рендерит страницу в
headless-браузере и читает реальные computed styles. Плагин превращает это в
фреймы, текст, картинки и SVG-иконки, сохраняя вложенность.

## Установка

### 1. Сервер (один раз)

```bash
cd ~/Desktop/html-to-figma-free/server
npm install          # поставит express + playwright и скачает Chromium
```

### 2. Запуск сервера (каждый раз перед импортом)

```bash
cd ~/Desktop/html-to-figma-free/server
node server.js       # поднимется на http://localhost:3789
```

### 3. Плагин в Figma (один раз)

1. Figma (desktop-приложение) → меню **Plugins → Development → Import plugin from manifest…**
2. Выбрать файл `~/Desktop/html-to-figma-free/plugin/manifest.json`
3. Плагин появится в **Plugins → Development → HTML to Figma (Free)**

## Использование

1. Убедись, что сервер запущен (`node server.js`).
2. В Figma запусти плагин, введи URL и ширину (1440 / 390 и т.п.), жми **Импортировать**.
3. Через несколько секунд на холсте появится фрейм со страницей.

## Что переносит хорошо

- Геометрию и вложенность блоков (позиции, размеры)
- Цвета фона, текста, бордеры, скругления, тени, прозрачность
- Текст (с размером, насыщенностью, межстрочным, выравниванием)
- Картинки (`<img>` и `background-image`) и inline-SVG иконки

## Ограничения (это MVP, не убийца html.to.design)

- **Шрифты:** если шрифта сайта нет в Figma — фоллбэк на Inter (структура и
  размеры сохраняются, гарнитура — нет).
- **Auto Layout** не реконструируется: всё на абсолютных координатах внутри
  вложенных фреймов (двигается группами, но без resize-правил).
- **CSS-градиенты** в фоне пропускаются (остаётся цвет).
- Очень тяжёлые страницы режутся по лимиту `MAX_NODES` (4000, см. `server.js`).
- `<canvas>`, видео, сложные `clip-path`/`mask`, псевдоэлементы — не переносятся.

## Тюнинг

- `server/server.js` → `MAX_NODES` — потолок числа нод.
- Viewport-высота рендера — 900px, дальше включается автоскролл для lazy-load.
- Хочешь импортировать страницу за логином/локалхост? Запускай Chromium
  не headless и добавь `storageState` — Playwright это умеет.
